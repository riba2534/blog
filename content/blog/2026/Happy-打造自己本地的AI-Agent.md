## 引言

[Claude Code](https://claude.com/product/claude-code) 最初是 Anthropic 的一个内部工程实验项目，于 2025 年 2 月作为 beta 版本首次公开发布（与 Claude 3.7 同期），并在 2025 年 5 月与 Claude 4 一起正式发布。它主打 **Agentic Coding（代理编程）**，是一款运行在终端（Terminal）中的 CLI 工具，能够理解代码库、通过自然语言命令执行日常任务、解释复杂代码、处理 git 工作流。由于其命令行特性，早期主要在极客和资深开发者圈子内传播。

随着 Claude 4.5 Sonnet 和 Opus 4.5（2025 年 11 月）的发布，模型能力大幅提升，底层推理能力显著增强。这款工具逐渐开始"出圈"，在一些非程序员圈子开始流行，包括银行职员、财务人员、法律从业者等需要使用计算机处理工作的群体。在 X 上每天有数十万条帖子，用户在分享自己如何使用 Claude Code。

**人和动物的根本区别是人可以使用工具，AI 也一样。Agent 和 Chatbot 的区别就在于 Agent 可以使用工具**。Claude Code 早已不只是一个 AI 编程工具，它是一个运行在自己计算机上的本地 AI Agent。因为它天生就运行在终端里，可以随意调用所有 Bash 工具，理论上来说，它可以完成计算机中的任何事情。

> Unix 哲学: Bash is All !

相比于 Manus 这类费用较高且效果不一定怎么样的 Agent，Claude Code（以下简称 CC）有如下优势：

- **本地环境运行**：天然继承自己工作环境的所有上下文（如常用的文件、正在处理的文档、表格等），而 Manus 的每一次任务都是新创建一个容器来执行。
- **工具更强大**：受益于本地系统的资源和环境，CC 可以调用的工具会更多。它甚至会在缺少某个工具时主动安装（比如你让它处理视频格式，它会自己下载安装 FFmpeg），而 Manus 只能使用内置工具。
- **执行与验证**：收集完任务信息后，CC 可以主动启动程序执行代码，验证自己的处理结果是否正确，最后给出答案。

但问题来了：Claude Code 只能在终端里用。你不可能随时都守在电脑前，有时候灵光一闪的想法需要有个东西来承接。最好能有一个 24 小时在线的 Agent，在手机上给它派个任务，让它在后台默默干活。

## Happy

最近在 GitHub 遨游的时候发现了一个项目：[Happy](https://github.com/slopus/happy)。简单来说，**Happy 是一个 Claude Code 的远程控制客户端**。它让你可以通过手机、平板或者网页浏览器，远程操控运行在你电脑上的 Claude Code。核心架构是：

```
你的手机/平板 <---> Happy 中转服务器 <---> 你的电脑上的 Claude Code
```

注意这里的关键点：**Claude Code 始终运行在你自己的电脑上**，而不是某个云服务器。这意味着：

- 你的代码不会上传到任何第三方服务器
- 你可以访问本地的所有文件和开发环境
- 现有的开发工具链和配置完全不受影响

Happy 的部署架构分为三部分：

1. **执行端**：一台电脑安装 Claude Code，作为真正执行任务的机器
2. **中转服务器**：官方免费提供，也可以用开源代码在公网服务器上自建
3. **客户端**：手机、平板、Web，通过中转服务器给 CC 下发任务

在我看来，Happy 解决了以下几个核心痛点：

1. **随时随地访问**：在手机上就能给 Claude Code 派任务，解放了对电脑的依赖
2. **多会话并行**：可以同时开多个 Claude Code 会话，验证不同的想法
3. **智能通知**：当 Agent 需要你确认或者任务完成时，会推送通知到你的手机
4. **完全免费**：开源项目，MIT 许可证

安装和使用很简单，在需要部署的设备上执行：

```bash
npm i -g happy-coder && happy
```

然后下载手机 App 就可以使用了。

所有内容都是端到端加密的，安全性可以放心。关于安全和隐私可以参考官方声明：[Privacy Policy](https://happy.engineering/privacy/)

Happy 的工作原理如下：

- **CLI 程序 (happy)**：运行在你的电脑上，启动 Claude Code 并监控其行为，将信息加密后发送到中转服务器
- **移动端 App**：运行在你的手机上，从服务器获取加密数据并展示 Claude Code 的工作状态
- **中转服务器 (Relay Server)**：连接你的电脑和手机，传递加密消息。服务器无法读取你的数据，只负责转发加密内容

## 我的实践

之前买了一个 Mac mini M4 做路由器，搬家之后从网络拓扑里面给摘掉了，吃灰了大半年。发现这个项目后，我给它装上了 Claude Code，改造成了位于家里的 Happy Server，显示器拔掉，就能作为 7×24 小时不间断运行的 AI Agent 服务器。

正好公网有台闲置的国内腾讯云服务器，部署了中转服务 [happy-server](https://github.com/slopus/happy-server)，国内网络访问应该能快一些。

部署好以后用了一下，感觉打开了新世界的大门。我发现它最大的用途不是写代码，而是**验证想法**。

> 你随时随地迸发出来的灵机一动，想法可行性立刻就可以得到验证。

举个例子：最近 Polymarket 在金融预测市场中很火，它把「对未来的判断」变成可交易资产。由于 Yes + No = 1 的机制，如果某个事件因为流动性不足或情绪极端，在短暂的窗口出现 Yes + No < 1 的情况，那就是一个无风险套利机会——同时买入 Yes 和 No，无论最终结果如何，收益都是确定的。X 上有很多类似的分享帖子，有人套利了几十上百万美金（现在这种机会几乎不存在了），而这种事情最适合 Agent 干。

但是作为普通人，如果你想验证这些想法，是有一定门槛的。假设你临时有一个不错的点子，而当时又不在电脑前，你可以很方便地用手机给 Agent 下发任务，让它通过 API 获取市场最新信息，通过计算告诉你是否存在机会。这会大大降低决策成本。
